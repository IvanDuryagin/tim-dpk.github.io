О проекте

«ТИМ-ДПК» — это программа с частично открытым исходным кодом, которая автоматически переносит данные об узлах соединений из BIM-моделей (Archicad, Revit) в форматы, понятные расчётным МКЭ-комплексам (ANSYS, SCAD).

Мы превращаем ручной, многочасовой процесс выявления и описания стыков ДПК-панелей в автоматизированный конвейер: IFC → JSON → готовый скрипт для расчёта. Цель — ускорить подготовку расчётных схем и исключить ошибки ручного ввода.

Алгоритм работы

Шаг 1. Экспорт BIM-модели в IFC

Пользователь экспортирует трёхмерную модель из Archicad, Revit или Renga в открытый формат IFC, сохраняя геометрию, слоистую структуру ДПК-панелей и данные о нагрузках.

Шаг 2. Загрузка IFC в ядро «ТИМ-ДПК»

Программа парсит IFC-файл, идентифицирует все CLT/ДПК-панели (объекты IfcPlate) и извлекает их геометрию, толщину, ориентацию слоёв и уникальные идентификаторы.

Шаг 3. Обнаружение и классификация узлов соединений

Алгоритм пространственного анализа находит места контакта панелей (пересечение ограничивающих параллелепипедов, общие грани) и определяет тип узла (угловой, Т-образный, крестовый, нахлёст).

Шаг 4. Формирование JSON-спецификации

На основе собранных данных создаётся структурированный JSON-файл — машиночитаемое техническое задание, содержащее все характеристики узлов, материалов и нагрузок.

Шаг 5. Генерация скрипта для целевого МКЭ-пакета

Конвертер преобразует JSON в APDL-скрипт для ANSYS, командный файл для SCAD/LIRA или скрипт для Rhino+Grasshopper.

Шаг 6. Расчёт в конечно-элементном комплексе

Инженер запускает сгенерированный скрипт в выбранной среде и получает готовую конечно-элементную модель с назначенными свойствами, контактами и нагрузками, после чего выполняет расчёт и анализирует результаты.
